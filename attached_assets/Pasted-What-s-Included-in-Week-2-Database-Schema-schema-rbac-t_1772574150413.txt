What's Included in Week 2
Database Schema (schema-rbac.ts)
✅ roles table - Application roles

5 standard roles: admin, underwriter, loanOfficer, borrower, auditor
Custom role support
Audit fields (createdAt, updatedAt)

✅ permissions table - Granular permissions

Resource:action format (deal:read, document:upload, user:delete)
Categorization for UI/documentation
50+ built-in permissions

✅ rolePermissions table - Many-to-many role-permission mapping

Fast permission lookups
Audit trail of who granted permissions

✅ userPermissions table - Direct user permission overrides

Temporary access grants
Optional expiration
Revocation tracking

✅ permissionDenials table - Immutable audit log

Every failed permission check logged
User, IP, endpoint, and reason recorded
Critical for SOC 2 compliance

RBAC Utilities (rbac.ts)
✅ Permission Cache (5-min TTL)

Reduces database queries 95%
Auto-invalidate on permission changes
Global or per-user cache busting

✅ Permission Checking Functions

hasPermission(userId, permissionName) - Single check
hasAnyPermission(userId, [...]) - OR logic
hasAllPermissions(userId, [...]) - AND logic

✅ Permission Middleware

requirePermission("deal:read") - Single permission
requireAnyPermission([...]) - OR logic
requireAllPermissions([...]) - AND logic
Automatic denial logging

✅ Built-in Permission Constants

DEAL_READ, DEAL_CREATE, DEAL_UPDATE, DEAL_DELETE, DEAL_APPROVE
DOCUMENT_READ, DOCUMENT_UPLOAD, DOCUMENT_DELETE
USER_READ, USER_CREATE, USER_UPDATE, USER_DELETE
ROLE_READ, ROLE_CREATE, ROLE_UPDATE, ROLE_DELETE
PERMISSION_GRANT, PERMISSION_REVOKE
ADMIN_ACCESS, AUDIT_READ

RBAC Routes (rbac.ts)
✅ Role Management

GET /api/rbac/roles - List all roles
POST /api/rbac/roles - Create new role

✅ Permission Management

GET /api/rbac/permissions - List all permissions
POST /api/rbac/permissions - Create new permission

✅ Role-Permission Assignment

POST /api/rbac/roles/:roleId/permissions/:permissionId - Grant
DELETE /api/rbac/roles/:roleId/permissions/:permissionId - Revoke

✅ User Permission Overrides

POST /api/rbac/users/:userId/permissions/:permissionId - Grant with reason & expiry
DELETE /api/rbac/users/:userId/permissions/:permissionId - Revoke with reason

✅ Reporting & Audit

GET /api/rbac/users/:userId/permissions - List user's permissions
GET /api/rbac/permission-denials - Audit trail of denied access
