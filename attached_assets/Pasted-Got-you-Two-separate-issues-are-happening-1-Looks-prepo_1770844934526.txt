Got you. Two separate issues are happening:
	1.	“Looks prepopulated but still must be typed”
That means you’re creating fillable fields in PandaDoc, but you’re not setting their value as a prefill (or you’re assigning the value to a property PandaDoc ignores). In PandaDoc, having text visible can be a placeholder/appearance thing, while the field still has an empty stored value.
	2.	Fields land slightly too high
That’s almost always a coordinate system mismatch:

	•	your app uses top-left origin, PandaDoc uses bottom-left (or vice versa), or
	•	you’re not converting using the true PDF page height (you’re using viewport height, zoomed height, or container height), or
	•	you’re not compensating for page padding/top offset in the rendered PDF canvas.

The fix: normalize coordinates + convert using PDF page dimensions, and set true prefill values.
