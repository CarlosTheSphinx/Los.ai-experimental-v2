# 2) Pricing formula to get final “price”

You have two “prices” in this program:

1. **Final Interest Rate**
2. **Final Leverage Caps (LTC/LTAIV/LTARV)**

I’ll give both formulas.

---

## A) Final Interest Rate formula

### Step 0: Determine eligibility first (if ineligible, stop)

Run disqualifiers (Section 3). If any disqualifier triggers:

* Output: `eligible = false`
* Output: `reasons = [ ... ]`
* Do not compute price

### Step 1: Base Rate

From table:

* Light Rehab: **9.25**
* Heavy Rehab: **9.50**
* Bridge (No Rehab): **9.25**
* GUC: **10.00**

So:

```
rate = baseRate[loanType]
```

### Step 2: Add all applicable RATE adjusters (stack)

From pricing matrix:

```
if isMidstream == true: rate += 0.25
if purpose == cash_out: rate += 0.50
if propertyType == multifamily: rate += 1.00
if fico < 700: rate += 0.25
```

### Step 3: Output final rate

```
finalRate = round(rate, 3)
```

✅ That is your interest rate calculator.

---

## B) Points formula (from earlier version)

If your points are constant:

* RTL points default = **2.0** (based on your original table)

Then:

```
points = 2.0
```

If you add tiered points later, you can model it the same way as rate adjusters.

---

## C) Final Leverage Caps formula (LTC/LTAIV/LTARV)

This is a separate “pricing track” but the same mechanism.

### Step 1: Start with base caps from the experience + loan type grid

Example (from earlier grid you shared):

* Institutional:

  * Light Rehab: LTAIV/LTC = 90/85, LTC 95/90, LTARV 75
  * Heavy: 85/85 etc.

So you pick a base cap set:

```
caps = baseCaps[experienceTier][loanType]
```

### Step 2: Apply leverage overlays and reductions (stack)

Overlays you’ve provided include:

* MSA reduction: -5% LTC (and/or others)
* FL bridge no rehab: -5% across leverage
* FICO < 680: -10% across LTC/LTAIV/LTARV + cash-out not allowed
* Declining market: -10% LTC and LTARV
* Bridge listed 120+ days: -5% leverage reduction (and requires 3rd party valuation)
* > 4 units: cap LTAIV/LTC at 80% + unit cap 20 + GUC not allowed + experienced only
* 0 experience: cap LTC at 30% budget/AIV
* GUC development project: cap LTC at 80%

So:

```
for each overlay that matches inputs:
  caps.ltc += overlay.deltaLTC
  caps.ltaiv += overlay.deltaLTAIV
  caps.ltarv += overlay.deltaLTARV
  caps.ltc = min(caps.ltc, overlay.capMaxLTC) if present
  ...
```

### Step 3: Clamp to sane bounds

```
caps = clamp(caps, 0, 0.95)
```

### Step 4: Output final caps

Return caps and list what reduced them.

# 3) Disqualifiers library (backend-ready)

This is the “rulebook” your backend runs first.
Each disqualifier has:

* `id`
* `condition` (machine)
* `message` (partner-friendly explanation)
* `severity` = `"ineligible"`

I’m going to include everything you’ve shown that is a **hard stop**, plus a few that are “flag-only” (not disqualifiers) at the end.

---

## A) Loan amount disqualifiers

### DQ-LOAN-001: Below minimum loan amount

* If `loanAmount < 125000` → ineligible
  Message: “Minimum loan amount is $125,000.”

### DQ-LOAN-002: Below minimum loan amount for GUC

* If `loanType == guc` AND `loanAmount < 150000` → ineligible
  Message: “Minimum loan amount for GUC is $150,000.”

### DQ-LOAN-003: Exceeds maximum loan amount

* If `loanAmount > 5000000` → ineligible
  Message: “Maximum loan amount is $5,000,000.”

---

## B) Credit disqualifiers

### DQ-CREDIT-001: FICO below minimum

* If `fico < 660` → ineligible
  Message: “Minimum FICO is 660.”

### DQ-CREDIT-002: No full guaranty requires higher FICO

* If `hasFullGuaranty == false` AND `fico < 700` → ineligible
  Message: “Without a full guaranty, minimum FICO is 700.”

### DQ-CREDIT-003: Mortgage history exceeds allowed lates

* If `mortgageLate60Last24 > 0` → ineligible
* If `mortgageLate30Last24 > 1` → ineligible
  Message: “Mortgage history must be 0x60 and no more than 1x30 in the past 24 months.”

### DQ-CREDIT-004: BK seasoning < 60 months

* If `monthsSinceBK != null` AND `monthsSinceBK < 60` → ineligible
  Message: “Bankruptcy must be seasoned 60+ months.”

### DQ-CREDIT-005: Foreclosure seasoning < 60 months

* If `monthsSinceForeclosure != null` AND `monthsSinceForeclosure < 60` → ineligible
  Message: “Foreclosure must be seasoned 60+ months.”

### DQ-CREDIT-006: Short sale / DIL seasoning < 60 months

* If `monthsSinceShortSaleOrDIL != null` AND `monthsSinceShortSaleOrDIL < 60` → ineligible
  Message: “DIL/Short sale must be seasoned 60+ months.”

## C) Experience-based disqualifiers

### DQ-EXP-001: No experience cannot do Heavy Rehab

* If `experienceTier == no_experience` AND `loanType == heavy_rehab` → ineligible
  Message: “Heavy Rehab is not available for no-experience borrowers.”

### DQ-EXP-002: No experience cannot do GUC

* If `experienceTier == no_experience` AND `loanType == guc` → ineligible
  Message: “Ground-up construction requires experienced or institutional sponsorship.”

### DQ-EXP-003: Cash-out Bridge not allowed for no experience

* If `experienceTier == no_experience` AND `loanType == bridge_no_rehab` AND `purpose == cash_out` → ineligible
  Message: “Cash-out bridge is not allowed for no-experience borrowers.”

### DQ-EXP-004: GUC requires 5+ completed projects if “Experienced”

* If `loanType == guc` AND `experienceTier == experienced` AND `completedProjects < 5` → ineligible
  Message: “GUC requires 5+ completed projects for experienced borrowers.”

### DQ-EXP-005: 0 completed projects leverage cap conflict (optional hard-stop)

You stated: “If borrower has 0 experience LTC maxed at 30% budget/AIV.”
This is technically not a disqualifier, but many deals become impossible.

Two ways:

* **Soft** (recommended): apply cap and keep eligible
* **Hard**: if requested LTC exceeds 30% you can mark ineligible

Soft logic:

* Not disqualified. Just cap leverage.
