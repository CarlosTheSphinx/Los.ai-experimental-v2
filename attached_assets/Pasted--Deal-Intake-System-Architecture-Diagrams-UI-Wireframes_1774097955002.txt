## Data Flow Diagram: Deal Submission to Analysis

```
                         BROKER
                            │
                            ↓
                    ┌─────────────────┐
                    │  Deal Form Submission
                    │  {loan_amount, asset_type,
                    │   property_info, borrower_info,
                    │   documents, ...}
                    └────────┬────────┘
                             │
                             ↓
                    ┌─────────────────────────────┐
                    │  VALIDATION LAYER
                    │  ├─ Check required fields
                    │  ├─ Validate data types
                    │  ├─ Confirm required docs
                    │  └─ If invalid → BLOCK
                    └────────┬────────────────────┘
                             │
                             ↓ [Valid]
                    ┌─────────────────────────────┐
                    │  CONDITIONAL RULES ENGINE
                    │  For each document rule:
                    │    if conditions_met:
                    │      required_docs.add(docs)
                    │  Produce required_docs list
                    └────────┬────────────────────┘
                             │
                             ↓
                    ┌─────────────────────────────┐
                    │  DOCUMENT REQUIREMENT CHECK
                    │  submitted_docs vs required
                    │  If missing: BLOCK + error
                    │  If complete: PROCEED
                    └────────┬────────────────────┘
                             │
                             ↓ [All docs present]
                    ┌─────────────────────────────┐
                    │  DEAL STORAGE
                    │  Save deal to database
                    │  Status = "submitted"
                    │  Store all document refs
                    │  Create version history
                    └────────┬────────────────────┘
                             │
                             ↓
                    ┌─────────────────────────────┐
                    │  AUTO-ROUTE TO LENDER
                    │  Queue for lender inbox
                    │  Send notification
                    └────────┬────────────────────┘
                             │
                             ↓
                           LENDER
                             │
          ┌──────────────────┼──────────────────┐
          │                  │                  │
          ↓                  ↓                  ↓
    ┌──────────────┐  ┌─────────────┐  ┌───────────────┐
    │   AGENT 1    │  │   AGENT 2   │  │   AGENT 3     │
    │   VALIDATE   │→ │   MATCH &   │→ │  GENERATE     │
    │   STRUCTURE  │  │   ANALYZE   │  │  FEEDBACK     │
    │              │  │             │  │               │
    │ Output:      │  │ Output:     │  │ Output:       │
    │ Structured   │  │ Matching    │  │ Key flaws     │
    │ deal JSON    │  │ report +    │  │ Confidence    │
    │ Validation   │  │ risk scores │  │ Verdict       │
    │ status       │  │             │  │ Fund recs     │
    └──────────────┘  └─────────────┘  └───────────────┘
          │                  │                  │
          └──────────────────┼──────────────────┘
                             │
                             ↓
                    ┌─────────────────────────────┐
                    │  STORE AI ANALYSIS RESULTS
                    │  Save all agent outputs
                    │  Index by deal_id
                    │  Create analysis_record
                    └────────┬────────────────────┘
                             │
                             ↓
                    ┌─────────────────────────────┐
                    │  LENDER DASHBOARD UPDATE
                    │  Display:
                    │  ├─ Deal summary
                    │  ├─ AI verdict & confidence
                    │  ├─ Key flaws
                    │  ├─ Fund recommendations
                    │  ├─ Document list
                    │  └─ Action buttons
                    └─────────────────────────────┘
```

## Fund Matching Logic Flowchart

```
┌─────────────────────────────────┐
│  Deal submitted with metrics:
│  • loan_amount = $2.5M
│  • asset_type = "Office"
│  • property_state = "CO"
│  • ltv_pct = 68%
│  • ltc_pct = 72%
└────────────┬────────────────────┘
             │
             ↓
    ┌────────────────────────┐
    │  Query all funds       │
    │  WHERE is_active=true  │
    │  (Result: 15 funds)    │
    └────────┬───────────────┘
             │
             ↓
    ┌─────────────────────────────────┐
    │  For each fund, check criteria: │
    └────────┬────────────────────────┘
             │
    ┌────────┴──────────────┬──────────────────┐
    ↓                       ↓                  ↓
FUND 1          FUND 2              FUND 3
Prime Capital   Growth Finance      Value Fund
                Partners
LTV: 70-75%     LTV: 65-80%         LTV: 50-70%
LTC: 75-80%     LTC: 70-85%         LTC: 60-75%
Min: $500k      Min: $1M            Min: $2M
Max: $10M       Max: $15M           Max: $8M
States: CA,CO,TX States: CA,CO,NY   States: CA,CO
Assets: MF,OF,RT Assets: ALL        Assets: MF,OF

Check:
1. LTV 68% in [70-75]? NO   YES              NO

2. LTC 72% in [75-80]? NO   YES              NO

3. Amt $2.5M in range? YES  YES              YES

4. State CO in list? YES     YES              YES

5. Asset Office OK? YES      YES              YES

MATCH?         ✗ NO        ✓ YES              ✗ NO
Score:         --          92%                --

    │
    └─────────────┬──────────┘
                  │
         ┌────────┴────────┐
         ↓                 ↓
    ELIGIBLE           NO MATCHES
    FUNDS              [2 funds match]
    [Growth Finance]   → Send to lender
                         "2 viable options"
         │
         ↓
    FUND MATCH
    COMPLETE
```

## AI Orchestration Sequence Diagram

```
BROKER SUBMITS DEAL
        │
        ↓
┌──────────────────────────────────────┐
│ AGENT 1: Deal Validator & Structurer │
├──────────────────────────────────────┤
│                                      │
│ Parse submission JSON                │
│   ├─ Extract all fields              │
│   ├─ Validate data types             │
│   ├─ Calculate metrics (LTV, LTC)    │
│   ├─ Check required fields           │
│   └─ Flatten for next agent          │
│                                      │
│ Output: structured_deal_json         │
│ Error handling: validation_errors[]  │
│                                      │
└────────────┬─────────────────────────┘
             │
             ↓
    [IF validation errors]
    → STOP, return errors to lender
    → Deal status = "validation_failed"

    [IF valid]
    → Continue to Agent 2

             │
             ↓
┌──────────────────────────────────────┐
│ AGENT 2: Fund Matcher & Analyzer     │
├──────────────────────────────────────┤
│                                      │
│ Receive structured_deal from A1      │
│                                      │
│ For each fund in database:           │
│   Check LTV in range?                │
│   Check LTC in range?                │
│   Check amount in range?             │
│   Check geography OK?                │
│   Check asset type OK?               │
│   → If all yes: ADD to eligible      │
│                                      │
│ Assess deal health:                  │
│   ├─ Borrower risk (credit, fin.)    │
│   ├─ Property risk (metrics, mkt)    │
│   ├─ Loan structure risk (LTV,DSCR)  │
│   ├─ Documentation risk (complete?)  │
│   └─ Score each 0-100                │
│                                      │
│ Output: matching_report              │
│         deal_health_assessment       │
│         eligible_funds[]             │
│                                      │
└────────────┬─────────────────────────┘
             │
             ↓
    [IF no eligible funds]
    → eligible_funds = []
    → Lender sees: "No viable funds"
    → Broker sees: "No matches"

    [IF eligible funds found]
    → Continue to Agent 3

             │
             ↓
┌──────────────────────────────────────┐
│ AGENT 3: Feedback Generator          │
├──────────────────────────────────────┤
│                                      │
│ Receive matching_report + A2 output  │
│                                      │
│ Identify key flaws:                  │
│   ├─ Policy violations?              │
│   ├─ Documentation gaps?             │
│   ├─ Borrower concerns?              │
│   ├─ Property red flags?             │
│   └─ Structure issues?               │
│                                      │
│ Calculate confidence_score:          │
│   60% fund_fit (how many matches?)   │
│   40% deal_health (avg risk score)   │
│   → score = 0-100                    │
│                                      │
│ Generate verdict:                    │
│   score > 75 → "PASS"                │
│   50-75     → "CONDITIONAL"          │
│   < 50      → "FAIL"                 │
│                                      │
│ Generate recommendations:            │
│   For each eligible fund:            │
│     - Match score                    │
│     - Why good fit                   │
│     - Any concerns                   │
│                                      │
│ Output: ai_feedback JSON             │
│         flaws[]                      │
│         confidence_score             │
│         verdict                      │
│         fund_recommendations[]       │
│                                      │
└────────────┬─────────────────────────┘
             │
             ↓
┌──────────────────────────────────────┐
│ STORE ANALYSIS RESULTS               │
│                                      │
│ Save to database:                    │
│   ├─ agent1_validation               │
│   ├─ agent2_matching                 │
│   ├─ agent3_feedback                 │
│   ├─ overall_verdict                 │
│   ├─ confidence_score                │
│   └─ timestamp                       │
│                                      │
└────────────┬─────────────────────────┘
             │
             ↓
    LENDER DASHBOARD SHOWS:
    • AI verdict & confidence
    • Key flaws + remediation
    • Strengths
    • Fund recommendations
    • All documents
    • Action options
```

## Configuration Hierarchy

```
SYSTEM ADMINISTRATOR
     │
     ├─ Configure global settings
     │  • Document storage limits
     │  • AI model versions
     │  • System features
     │
     ↓
LENDER
     │
     ├─ Create funds
     │  • Fund name, contact
     │  • Base criteria (LTV, amount, geography, assets)
     │  • Base document requirements
     │
     ├─ Define rules
     │  • Conditional document rules
     │  • Complex nested conditions
     │
     ├─ Create fund managers
     │  • Assign to specific funds
     │  • Grant customization permission
     │
     └─ Distribute deals
        • Choose fund(s) to submit to
        • Send to specific contacts
        • Track submissions
        │
        ↓
    FUND MANAGER (optional)
         │
         ├─ Customize fund criteria
         │  • Tighten LTV/LTC limits
         │  • Add net worth requirements
         │  • Restrict geographies
         │
         ├─ Modify document requirements
         │  • Add required docs
         │  • Remove optional docs
         │  • Add fund-specific questions
         │
         └─ (Cannot see/manage other funds)

BROKER
     │
     ├─ Submit deals
     │  • Fill deal form
     │  • Upload documents (conditional based on rules)
     │  • See their own deals
     │  • Cannot see AI feedback (unless no-match)
     │
     └─ (No configuration access)
```

This concludes the detailed architecture and UI wireframes for the commercial deal intake system.
